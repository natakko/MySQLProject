# Аналитическая хранимая процедура для MS SQL Server

Реализация отчета по продажам с расчетом бизнес-показателей и динамической фильтрацией данных.

## Описание решения
Проект представляет собой хранимую процедуру `sp_report_1`, предназначенную для формирования детальной отчетности по движению товаров. В коде реализованы механизмы агрегации данных, фильтрации и расчета производных финансовых показателей.

## Ключевые технические особенности
- **Динамический парсинг входных параметров:** Реализован алгоритм разделения строки (String Split) через цикл `WHILE` и функции `CHARINDEX/SUBSTRING`. Это позволяет передавать список групп товаров через запятую в одном параметре.
- **Работа с типами данных:** Выполнено преобразование формата `DATE` в целочисленный ключ `INT` (ггггммдд) для обеспечения корректной связи с таблицей фактов в хранилище данных.
- **Расчет аналитических показателей:**
    - Вычисление средней цены закупки, маржи и наценки с исключением НДС (ставка 20%).
    - Использование оконных функций (`OVER`) для определения доли продаж конкретной номенклатуры в общем обороте.
- **Обеспечение отказоустойчивости:** Применение функции `NULLIF` в операциях деления для предотвращения ошибок при обработке нулевых значений.

## Используемый стек технологий
- T-SQL (Microsoft SQL Server).
- Временные таблицы (`#tempdb`) для промежуточного хранения фильтров.
- Оконные функции и агрегатные конструкции.
- Реляционные связи (INNER JOIN) между таблицами фактов и справочниками (Goods, Stores, Date).

## Инструкция по развертыванию и запуску

### 1. Создание процедуры
Необходимо выполнить скрипт создания процедуры в контексте целевой базы данных (например, `TestTaskDB`).

### 2. Пример вызова процедуры
Для формирования отчета по конкретной группе товаров за определенный период используется следующая команда:
